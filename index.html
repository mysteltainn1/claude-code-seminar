Why CLI

챗봇 vs CLI

기존 방식

챗봇

  • 💬대화만 가능
  • 📋복사-붙여넣기 필요
  • 🔒내 파일에 접근 불가
추천

CLI

  • 💻내 컴퓨터에서 직접 실행
  • 📁파일 자동 생성/수정
  • 🧠프로젝트 전체를 이해
"챗봇은 레시피를 알려주는 셰프,
CLI는 직접 요리해주는 셰프"

09 / 16
Options

Claude 실행 옵션 알아두기

1 claude --resume — 이전 대화를 이어서 계속 작업. 터미널을 닫았다 켜도 OK
2 claude --dangerously-skip-permissions — 매번 승인 안 눌러도 됨 (자동 실행)
3 claude --dangerously-skip-permissions --resume — 둘 다 합치면 가장 편한 작업 모드
4 /model — 실행 중 모델 변경. opus 최신 모델로 맞추면 최고 성능
# 실전에서 가장 많이 쓰는 조합
$ claude --dangerously-skip-permissions --resume

# 실행 후 모델을 opus로 변경
> /model opus
💡 생산성을 위해 --dangerously-skip-permissions는 항상 켜두길 권장합니다. 진짜 위험한 건 옵션을 켜둬도 물어봐요.

Save

잠깐, 세이브 하셨나요?

만든 코드를 GitHub에 올려두면 어디서든 다시 쓸 수 있습니다. 게임으로 치면 세이브 포인트!

1 github.com 가입 — 무료 계정이면 충분
2 새 저장소(repository) 만들기 — Private으로 설정
3 Claude에게 한마디: "내 코드를 GitHub에 올려줘"
# Claude에게 이렇게 말하면 끝
> 이 프로젝트를 GitHub에 올려줘. private repo로 만들어줘.

# Claude가 알아서: git init → repo 생성 → push
✓ Repository created · Code pushed to GitHub
🔒 Private = 나만 볼 수 있음. 회사 프로젝트는 반드시 Private 또는 nimbleneuron-sandbox org에 올리세요.

  • 💻인터랙티브 도구, 대시보드
  • 🌐your-app.nimbleneuron.dev
  • 🔄GitHub 머지 → 자동 배포
# Pages 등록 — Claude에게 링크 주고 시키면 끝
> 이 파일을 사내 Pages에 올려줘
> https://github.com/nimbleneuron-sandbox/pages.git

# 웹앱 배포 — Coolify 흐름
Local → nimbleneuron-sandbox/your-repo → Coolify → *.nimbleneuron.dev

Caution

주의사항 & 팁

  • 🔐 시크릿은 코드에 넣지 않기 — API 키, 비밀번호는 환경변수로 분리
